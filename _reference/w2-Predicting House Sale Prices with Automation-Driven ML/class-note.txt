Artificial Intelligence isn’t just revolutionizing business. It’s also changing how we, as data professionals, do our work or learn about new technologies. Whether you're a data analyst or a data scientist, AI tools are already increasing productivity across every stage of the data pipeline.
Let’s take a closer look at how at a few stages of a typical data science project and at some of the AI-based solutions that come handy!
1. Automated Exploratory Data Analysis (EDA)
AI can now scan through datasets and generate key insights, summary statistics, and even visuals with minimal effort. Instead of manually writing dozens of lines of code to understand your data, AI tools like Sweetviz or YData Profiling (formerly pandas-profiling) can handle the heavy lifting, giving you a clear overview in seconds.

It all results in really fast prototyping and solving real business problems in a matter in matter of minutes! Similar to  SweetViz and YData Profiling, we’ll touch both Featuretools, AutoFeat and  H2O or PyCaret later in the course!

Modern AI-powered coding assistants (like GPT-based tools) can generate, correct, and refactor code based on natural language prompts. From writing a preprocessing function to debugging an error, AI helps you code faster and more accurately, and it keeps getting better.

Note on PandasAI and Google Colab
There are a lot of dependency issues when using the pandasai library in Google Colab (much fewer in a local Jupyter Notebook).
To keep things stable, we always start the notebook with: !pip uninstall -y pandasai. This way, if you “Run all” cells, Colab will first remove any old version of pandasai and then reinstall the correct versions of all required libraries. This avoids issues.
If you run some pip install commands in a notebook and they work the first time, but running the same commands again later suddenly breaks, just restart the kernel/runtime — this usually fixes conflicts.
Other useful tips:
Always install required libraries in the first cell, before importing anything. Otherwise Colab may require a restart because old versions are already loaded in memory.
Sometimes pandasai tries to downgrade pandas (to 1.5.3), which conflicts with Colab. Thats why below we pin pandas>=2.2.2 and install pandasai with --no-deps to avoid this.
If you see numpy.dtype size changed errors, restart the runtime.

Automated EDA with YData Profiling
What is Automated EDA?
Exploratory Data Analysis (EDA) is the process of summarizing the main characteristics of a dataset, often with visual methods. It helps data scientists spot anomalies, understand data distributions, and uncover relationships between variables.
Automated EDA tools speed up this process by generating interactive reports with descriptive statistics, correlations, missing value summaries, and visualizations — all with minimal code.
Why Ydata Profiling?
ydata-profiling is one of the most popular automated EDA libraries in Python. It stands out because:
It works directly with pandas DataFrames.
Generates a full EDA report in a single line.
Includes warnings and alerts about data issues.
Offers interactive HTML output for deeper analysis.
Does it sound exciting? If so, let’s jump right in and use ydata-profiling to perform fully automated EDA on the California housing dataset. Below, we’ll go step by step explaining how to handle EDA with ydata-profiling end to end. Let’s go!

Feature Engineering with Featuretools
What is Feature Engineering?
Feature engineering is the process of transforming raw data into meaningful features that improve the performance of machine learning models. This includes creating new variables, aggregating values, and capturing relationships in your data.
Feature engineering can be done either manually (like extracting a month or a day of the week from a date) or automatically via some advanced tools. There are a bunch of tools out there on the market but in this lesson we are going to focus on the one that really stands out. It is called Featuretools.
 
notion image
 
Featuretools is an open-source Python library for automated feature engineering, meaning that it does the work on its own! It excels at:
Handling relational datasets (multiple tables with relationships)
Automatically generating deep features through a process called Deep Feature Synthesis (DFS)
Saving time while creating structured, model-ready features
Unlike manual feature engineering or libraries (like pandas  or datetime) Featuretools automatically finds and builds potentially useful features using relationships between tables.
Why does it important? Well, because most of the data you’ll encounter “live” in some kind of tables in a database and these tables related to each other somehow.
Let’s move to some technical things now and look at the actual implementation.

Introducing Autofeat
Intro
So, we already know about and even practiced Featuretools as a tool for automated features generation. As you can guess, it isn’t the only Python library available for the purpose of features generation. 
Autofeat is a Python library that also does the job. It works great on flat, numeric datasets and generates new features by:
Applying mathematical transformations (e.g., log, sqrt, square)
Creating feature interactions (e.g., feature1 * feature2)
Selecting only the most useful and non-redundant features for modeling
It’s ideal when you're working with standard tabular data and want a fast, scikit-learn-friendly way to boost model performance.
Autofeat vs. Featuretools
Before we jump in, let’s spend some time comparing the two tools for automated features generation we learn about. Obviously, both Autofeat and Featuretools generate new features automatically — but their approach and ideal use cases are different. Here are the key differences:
Feature
Autofeat
Featuretools
Input Format
Single table with numeric data
Relational data (multiple linked tables)
Feature Strategy
Math transforms + interactions
Group-based aggregations + hierarchies
Time Awareness
❌ No
✅ Yes
Output Format
NumPy/pandas + scikit-learn compatibility
pandas DataFrame
Use Case
Flat tabular ML tasks
Structured/time-aware ML tasks
To sum things up, we can conclude that Autofeat is best when you want to enhance flat numeric data for regression or classification. Featuretools is better for relational, multi-table datasets.
Since we now know that Autofeat exceeds in features generation for a single table, let’s employ it now and generate features for the dataset that we’ve already worked with - the California Housing Dataset.

Automated ML with H2O.ai
Intro

Although we’ve learned a lot, there is still one stage that we haven’t automated yet. It is a stage of finding the best machine learning model and training it to achieve the best possible performance. Let’s see now what kind of automation are applicable there!
Introducing AutoML?
AutoML (Automated Machine Learning) is a process that automates the end-to-end pipeline of applying machine learning to real-world problems. This includes:
Data Preprocessing
Selecting the best models
Tuning hyperparameters for them
Stacking and ensembling to achieve even greater performance
AutoML is ideal when you want to quickly build high-quality models without writing the entire pipeline manually — and it’s also a great way to establish strong baselines.
There are multiple Auto ML tools available out there on the market, but today we are going to talk about the one that comes with H2O.ai

Introducing PyCaret
We’ve already seen how H2O AutoML helps us automate machine learning tasks. Now it’s time to explore another practical and beginner-friendly tool: PyCaret.
What Is PyCaret?
PyCaret is an open-source, low-code machine learning library in Python that helps automate tasks like:
Data Preprocessing
Model selection
Hyperparameter tuning
Model evaluation and interpretation
It’s built on top of powerful libraries like scikit-learn, XGBoost, and LightGBM, and wraps them in a unified, easy-to-use interface. PyCaret is especially useful when you want to experiment quickly and build solid baselines with minimal code.
