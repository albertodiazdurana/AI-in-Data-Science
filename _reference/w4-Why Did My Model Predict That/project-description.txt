Project: Why Did My Model Predict That?
Project Overview
You will work with a real-world medical dataset to predict the likelihood of heart disease in patients based on clinical features. Your main focus will be on explaining the model’s decisions using SHAP and LIME.

This is the third and final project in the AI Enhanced Productivity sequence. While the previous projects focused on automation and language-based analysis, this project emphasizes trust, transparency, and interpretability in machine learning models — especially in high-stakes domains such as healthcare.

About the Dataset
The data comes from Kaggle. The original dataset is called ”Heart Disease UCI Dataset” and is available on this competition page: https://www.kaggle.com/datasets/redwankarimsony/heart-disease-data. We uploaded the dataset to the Google Drive. You can access and download it using this link: https://drive.google.com/file/d/1uaDXpc5sS6W1BWGGepHfBZvTFgPM8gjG/view?usp=sharing.
The data itself has 303 patient records with features such as age, sex, chest pain type, resting blood pressure, cholesterol, fasting blood sugar, etc.
The target variable is stored in the num column and shows (1 - 4) if there is a presence of heart disease or (0) there is absence.
This dataset is small enough for interactive explanation tools but rich enough for meaningful insights.
What You’ll Do
In this project, your focus is not only on prediction accuracy, but on understanding and explaining model behavior.

Let’s briefly go over the project requirements, so you clearly know what to go over!
Preparation:
Create a new Colab Notebook that you’ll be working with for this project 
Data Exploration and Preprocessing:
Load and explore the dataset
Clean the data (e.g., handle missing values if any, encode categorical variables)
Split into training/testing sets
Model Training:
Train at least one model (e.g., Random Forest or XGBoost)
Evaluate performance with accuracy, precision, recall, and AUC
SHAP Analysis:
Use shap.TreeExplainer or appropriate SHAP explainer
Include:
Summary plot
Bar chart
Waterfall plot for individual predictions
Write your thoughts in a markdown cell what features are most impactful and why
LIME Analysis:
Use LimeTabularExplainer
Generate explanations for at least two different test cases
Compare to SHAP explanations
Business Interpretation:
When you finished the technical part of the project, we urge you to think about the results from the business perspective. To be concrete, we want to ask you to:
Identify which clinical features most strongly influence predictions
Reflect on how explanations may affect trust in the model (especially in healthcare)
Discuss when and why you might rely more on SHAP vs. LIME
