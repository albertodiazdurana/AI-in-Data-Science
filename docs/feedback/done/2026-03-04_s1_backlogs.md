## Proposal: Environment Management Protocol

### Problem

DSM does not currently mandate or guide environment setup (virtual environments,
dependency tracking) as part of project initialization or planning. Each project
requires a unique combination of libraries, yet there is no protocol for:

1. Identifying and documenting required packages during planning
2. Generating a `requirements.txt` (or equivalent) as a project artifact
3. Creating a project-specific virtual environment (`.venv`)
4. Scaling to more sophisticated tools (Poetry, conda, etc.) for larger projects

This gap surfaced during w2 planning: the plan included tool research and notebook
structure but omitted the foundational step of environment definition. Without a
requirements file, reproducibility — a core quality standard — is undermined.

### Proposed Solution

Add an **Environment Management** section to DSM project initialization that scales
by project complexity:

| Project Size | Minimum Requirement |
|-------------|-------------------|
| Small (single notebook) | `requirements.txt` in repo root |
| Medium (multi-notebook, shared libs) | `requirements.txt` + documented `.venv` setup |
| Large (application, multi-contributor) | `pyproject.toml` with Poetry/PDM or `environment.yml` for conda |

**For the planning phase:** project plans should include a step to define dependencies
and generate the appropriate environment file.

**For session start:** DSM session-start checks could verify that a requirements file
exists and that the local environment matches it (or flag drift).

### Evidence

- W2 plan for AI-in-Data-Science omitted venv/requirements despite identifying 6+
  libraries (ydata-profiling, sweetviz, autofeat, featuretools, pycaret, shap)
- The reference notebook pattern (disaster-tweets) includes inline installs but no
  requirements file, meaning dependency versions are implicit
- Colab-compatible notebooks install packages in cells, but local development needs
  a persistent environment — these are complementary, not alternatives
