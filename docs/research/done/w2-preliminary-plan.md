# Preliminary Plan: Predicting House Sale Prices with Automation-Driven ML

**Project:** AI-in-Data-Science (AI Enhanced Productivity, Project 1 of 3)
**Dataset:** Ames Housing (~1,500 residential homes, Ames, Iowa)
**Target:** SalePrice (regression)
**Environment:** Google Colab notebook
**Date:** 2026-02-22

---

## Objective

Predict house sale prices using the Ames Housing dataset, with emphasis on automating
as much of the ML workflow as possible. The project evaluates how automation tools
affect productivity, decision-making, and model quality.

## Phases

### Phase 1: Setup and Data Acquisition

- Obtain the Ames Housing dataset (available via Kaggle or OpenML)
- Install required libraries: sweetviz, ydata-profiling, featuretools, autofeat, h2o/pycaret
- Load data, verify shape, check dtypes and target distribution
- Document initial observations (feature count, data types, missing value overview)

### Phase 2: Automated EDA

- Generate YData Profiling report (full statistical profile, alerts, correlations)
- Generate SweetViz report (visual overview, target analysis with SalePrice)
- Compare the two reports: what each tool surfaces that the other misses
- Document: how automated EDA informed understanding of the dataset
- Identify data quality issues flagged by the reports (missing values, high cardinality,
  skewed distributions, potential outliers)

### Phase 3: Data Preprocessing

- Handle missing values (strategy informed by EDA reports)
- Encode categorical variables
- Address outliers if flagged by automated EDA
- Split into train/test sets (80/20, stratified if needed)

### Phase 4: Automated Feature Engineering

- Apply Featuretools if relational structure can be created (e.g., grouping by
  neighborhood, zoning)
- Apply Autofeat for mathematical transformations and feature interactions on numeric
  columns
- Compare: features generated by each tool, overlap, and complementarity
- Select final feature set (automated selection + manual review of relevance)
- Document: why certain generated features were relevant or useful

### Phase 5: Automated Model Training

- Option A: H2O AutoML (train multiple algorithms, auto-ensemble, leaderboard)
- Option B: PyCaret (low-code comparison of multiple models, auto-tuning)
- Run both if time permits; compare leaderboards
- Evaluate on test set: RMSE, MAE, R-squared
- Document: model performance, which algorithms performed best, ensemble impact

### Phase 6: Evaluation and Interpretation

- Compare automated pipeline results against a simple baseline (e.g., linear regression
  with manual features)
- Feature importance analysis (SHAP or model-native importance)
- Residual analysis: where does the model fail?
- Document: how model performance was evaluated and compared

### Phase 7: Reflections on Automation

- What automation improved or simplified in the workflow
- Where automation introduced limitations or required human judgment
- Key insights gained from relying on automated ML tools
- Time comparison: estimated manual effort vs automated pipeline
- Quality comparison: did automation produce better or worse results than a manual approach?

## Deliverables

1. Google Colab notebook with:
   - Technical artifacts (EDA reports, generated features, model leaderboards, metrics)
   - Analytical reasoning (markdown cells explaining decisions at each stage)
   - Reflections on automation (dedicated section)
2. Presentation materials (after all 3 AI Enhanced Productivity projects)

## Tool Selection Rationale

| Stage | Primary Tool | Backup / Complement |
|-------|-------------|---------------------|
| EDA | YData Profiling | SweetViz (comparison focus) |
| Feature Engineering | Autofeat (flat numeric data) | Featuretools (if relational structure created) |
| Modeling | H2O AutoML (robust, auto-ensemble) | PyCaret (low-code, fast comparison) |

## Risks and Mitigations

| Risk | Mitigation |
|------|-----------|
| Dependency conflicts in Colab | Pin versions as per class notes; install in first cell |
| H2O memory issues in Colab | Limit max_runtime_secs, reduce dataset if needed |
| Autofeat slow on full dataset | Subsample for feature engineering, apply to full for training |
| Featuretools requires relational structure | Create entity set from single table using grouping columns |
